The Problem

What agents actually encounter

These are not edge cases. They are the default behavior of most CLI tools today.

§10 · Critical · Interactivity

Pager blocks forever

The tool pages output and waits for a keypress. In a non-TTY pipe, the agent never receives a response.

# Agent calls a list command
$ kubectl get pods
↳ opens `less`, waits for input
↳ pipeline stalls — forever
§1 · Critical · Exit Codes

exit 1 means nothing

Every error returns exit 1. The agent can't tell if it's safe to retry, or if a partial write already happened.

# Deployment conflict — safe to retry?
$ deploy --env staging
Error: deployment locked
exit 1
# Half-deployed infra — also exit 1
$ deploy --env staging
Error: timeout after 30s
exit 1
§43 · Critical · Output Size

Unbounded output fills context

A list command dumps 10,000 records. The agent's context window fills up before it can act.

# No default limit, no pagination
$ aws s3 ls s3://my-bucket --recursive
↳ 847,293 lines of output
↳ context window: exhausted
↳ cost: $4.20 in input tokens
§69 · High · Argument Order

Flags silently misparsed

The parser treats --output as a positional argument. Exit code: 0. The agent receives plain text it can't parse.

# Natural LLM ordering
$ tool list users --output json
↳ "--output" → positional arg value
↳ returns ANSI table, not JSON
↳ exit 0 — no signal to retry
The Specification

Three layers. 147 requirements.

Structured so framework authors, command authors, and application builders each know exactly what they own.

REQ-F Framework-Automatic Enforced by the framework — command authors get these for free. Exit code table, response envelope, signal handling, ANSI suppression, pager blocking, non-TTY detection. 78 reqs
REQ-C Command Contract Declared by command authors at registration. Exit codes emitted, danger level, dry-run support, idempotency key, input/output schema. 28 reqs
REQ-O Opt-In Explicitly enabled by the application. --output json, --fields, --limit, tool manifest, --schema, --dry-run, --confirm-destructive. 41 reqs
Three core contracts
🔢

Exit Code Table

14 named codes (0–13) with machine-readable guarantees per code: retryable: true/false, side_effects: "none" | "partial" | "complete". An agent receiving exit 7 (CONFLICT) knows it's safe to retry. Receiving exit 6 (PARTIAL_FAILURE) knows to inspect state first.

📦

Response Envelope

Every command wraps output in {"ok", "data", "error", "warnings", "meta"}. Same keys on success, failure, and empty results. Agents never parse free-text to determine what happened.

🗺️

Tool Manifest

tool manifest --output json returns the complete command tree: every subcommand, flag, type, exit code map, and example. One call replaces O(N) --help iterations across all subcommands.
